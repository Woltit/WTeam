# Інструкція користувача — RentGo

**RentGo** — двостороння peer-to-peer платформа для короткострокової оренди речей в Україні. Платформа з'єднує **власників**, які хочуть здавати своє майно в оренду, з **орендарями**, яким річ потрібна на обмежений час (дриль, намет, камера, велосипед тощо).

Ця інструкція складається з двох частин:

1. **Встановлення та розгортання** — як підготувати середовище й запустити продукт.
2. **Використання** — повний опис усіх функцій і можливостей платформи.

> 💡 **Швидкий доступ.** Платформа вже задеплоєна — користуватися нею можна без локального встановлення, просто відкривши у браузері:
> **[https://w-team-sooty.vercel.app](https://w-team-sooty.vercel.app/)**
> Детальніше — у розділі [1.8. Онлайн-доступ](#18-онлайн-доступ-задеплоєна-версія).

# Частина 1. Встановлення та розгортання

Продукт складається з трьох частин:

- **Бекенд** — Java Spring Boot (REST API на `http://localhost:8080`, базовий шлях `/api/v1`);
- **Фронтенд** — React + TypeScript (Vite, за замовчуванням `http://localhost:5173`);
- **Інфраструктура** — PostgreSQL та Apache Kafka, які піднімаються через Docker Compose.

## 1.1. Системні вимоги

| Компонент | Версія | Призначення |
|---|---|---|
| **Docker + Docker Compose** | актуальна | Запуск PostgreSQL та Kafka |
| **JDK (Temurin)** | 25.0.1 | Збірка та запуск бекенду |
| **Node.js** | 22+ | Запуск фронтенду |
| **pnpm** | 11+ | Менеджер пакетів фронтенду |
| **Git** | актуальна | Отримання вихідного коду |

> Версію Java зафіксовано у файлі `backend/.tool-versions` (`java temurin-25.0.1`). Рекомендується використовувати менеджер версій (наприклад, `asdf` або `sdkman`).

## 1.2. Отримання вихідного коду

```bash
git clone <URL-репозиторію> WTeam
cd WTeam
```

Структура проєкту:

```text
WTeam/
├── backend/      # Spring Boot API
├── frontend/     # React SPA
└── README.md
```

## 1.3. Налаштування змінних оточення

Бекенд читає налаштування з файлу `backend/.env`. Створіть або відредагуйте його, заповнивши ключові параметри:

```dotenv
# Профіль запуску
SPRING_PROFILES_ACTIVE=dev

# База даних (локальний PostgreSQL із docker compose)
DB_HOST=localhost
DB_PORT=5432
DB_NAME=rentgo
DB_USERNAME=postgres
DB_PASSWORD=postgres

# JWT
JWT_SIGNING_KEY=<довгий-секретний-ключ>
JWT_EXPIRATION=900000

# Google OAuth2 (опційно, для входу через Google)
OAUTH2_CLIENT_ID=<client-id>
OAUTH2_CLIENT_SECRET=<client-secret>

# AI-асистент
OPENAI_API_KEY=<ключ>

# Поштові сповіщення (опційно)
MAIL_USERNAME=<email>
MAIL_PASSWORD=<пароль-застосунку>

# Хмарне сховище зображень (Cloudinary) — для завантаження фото речей
CLOUDINARY_CLOUD_NAME=<cloud-name>
CLOUDINARY_API_KEY=<api-key>
CLOUDINARY_API_SECRET=<api-secret>
```

> ⚠️ Файл `.env` містить секрети — зберігайте його обережно. Параметри `OAUTH2_*`, `OPENAI_API_KEY`, `MAIL_*` та `CLOUDINARY_*` опційні: без них працюватиме все, крім відповідної функції (вхід через Google, AI-асистент, поштові сповіщення, завантаження фото речей).

## 1.4. Запуск інфраструктури (PostgreSQL, Kafka)

З каталогу `backend/` підніміть контейнери:

```bash
cd backend
docker compose up -d
```

Буде запущено два контейнери:

- `wteam_container` — PostgreSQL 17 на порту `5432`;
- `wteam_kafka_container` — Apache Kafka на порту `9092`.

Перевірити стан:

```bash
docker compose ps
```

## 1.5. Запуск бекенду

З каталогу `backend/`:

```bash
./gradlew bootRun
```

Під час першого запуску застосунок виконає міграції бази даних (схема + початкові дані, зокрема категорії та тестовий адміністратор). Після старту API доступне за адресою `http://localhost:8080/api/v1`.

![Скріншот 2](docs/screenshots/02-backend-run.png)

*Скріншот 2. Лог успішного старту бекенду в терміналі (рядок про запуск на порту 8080).*

## 1.6. Запуск фронтенду

У новому терміналі з каталогу `frontend/`:

```bash
cd frontend
pnpm install
pnpm dev
```

Vite запустить застосунок за адресою `http://localhost:5173`. Фронтенд звертається до бекенду за адресою з `frontend/src/constants.ts` (`API_BASE = http://localhost:8080/api/v1`).

# Частина 2. Використання платформи

## 2.1. Загальний інтерфейс

Угорі кожної сторінки розташовано **навігаційну панель**:

- **Логотип RentGo** — повернення до каталогу;
- **Каталог** — перегляд усіх доступних речей;
- **AI-помічник** — підбір речей за описом ситуації;
- **Чати**, **Мої бронювання**, **Мої речі**, **Додати оголошення**, **Профіль** — доступні після входу;
- **Адмін** — лише для адміністраторів;
- **Перемикач мови UA / EN** — інтерфейс підтримує українську та англійську;
- **Перемикач теми** — світла / темна тема;
- **Вхід / Реєстрація** або ім'я користувача та кнопка **Вийти**.

## 2.3. Каталог і пошук речей

Каталог — стартова сторінка платформи. Тут можна:

- **Шукати за назвою** — поле пошуку;
- **Фільтрувати за містом** — поле введення міста;
- **Фільтрувати за категорією** — клік на кнопку категорії (наприклад, Туризм, або **Усі** для скидання фільтра);
- **Гортати сторінки** — пагінація внизу списку.

Кожна картка речі показує зображення, назву, ціну за день (₴) та за тиждень.

## 2.4. Сторінка речі

Клік на картку відкриває детальну сторінку, де відображено:

- **Галерею фотографій**: велике головне зображення та мініатюри решти фото. Клік на фото відкриває його на весь екран (lightbox) із перегортанням стрілками вперед/назад;
- **Стан речі** (ідеальний, хороший тощо), позначку **✓ Перевірено** для верифікованих речей;
- **Ціну за день і за тиждень**, суму **застави** (депозиту);
- **Опис** і **теги**;
- Блок **Власник**: ім'я, рейтинг довіри (⭐ Trust Score) та кількість успішних оренд;
- **Відгуки про власника**;
- Блок **бронювання** з календарем доступності.

Якщо ви — власник цієї речі, замість форми бронювання відображаються кнопки **Редагувати** та **Видалити**.

## 2.5. Бронювання та календар доступності

На сторінці речі (для авторизованого користувача, що не є власником):

1. У **календарі доступності** виберіть діапазон дат — початок і кінець оренди. Зайняті дати позначені й недоступні для вибору, що захищає від **подвійного бронювання**.
2. Календар автоматично порахує орієнтовну вартість (ціна за день × кількість днів) та суму застави.
3. Натисніть кнопку бронювання (**Забронювати та написати**) — створюється запит на оренду й відкривається чат із власником.

Якщо ви не авторизовані, замість форми буде запрошення увійти або зареєструватися.

## 2.7. Мої оголошення

Сторінка **Мої оголошення** показує сіткою всі оголошення, які ви додали як власник. Для кожної речі видно стан, статус верифікації (перевірено / на розгляді), місто та ціну. Клік на картку відкриває сторінку речі, де її можна редагувати або видалити. Якщо оголошень ще немає, сторінка пропонує одразу **Додати оголошення**.

## 2.8. Мої бронювання (орендар і власник)

Сторінка **Мої бронювання** має дві вкладки:

- **Оренди** — бронювання, які ви створили як **орендар**;
- **Пропозиції** — запити на ваші речі, де ви — **власник**.

Усередині кожної вкладки є фільтри за статусом (наприклад, **Усі**, **Очікують**).

Доступні дії залежно від ролі та статусу:

- **Власник** може **підтвердити** або **відхилити** запит, а після завершення оренди — позначити її **завершеною**;
- **Орендар** може **скасувати** своє бронювання, а для підтвердженого бронювання — **оплатити** його (див. наступний розділ).

Після кожної дії з'являється підтвердження.

## 2.9. Оплата бронювання (LiqPay)

Оплата виконується орендарем після підтвердження бронювання власником:

1. На сторінці **Мої бронювання** у відповідному бронюванні натисніть **Оплатити (LiqPay)**.
2. Система створює платіж і перенаправляє вас на платіжну сторінку.
3. Після підтвердження ви повертаєтесь на сторінку **Мої бронювання** з оновленим статусом.

## 2.11. AI-асистент

Сторінка **AI-помічник** підбирає речі за вільним текстовим описом ситуації:

1. Опишіть свою потребу в полі вводу (наприклад, *«їду на три дні в Карпати»*) або скористайтеся прикладами-підказками.
2. Натисніть кнопку запиту.
3. Асистент поверне рекомендовані речі з каталогу. Якщо підходящих позицій немає — буде запропоновано переформулювати запит.

## 2.12. Профіль і верифікація

Сторінка **Профіль** дозволяє:

- Завантажити та змінити **аватар**;
- Заповнити дані: **ім'я**, **прізвище**, по батькові, **дату народження**, **телефон**;
- Подати **запит на верифікацію особистості** (ручна перевірка адміністратором);
- Переглянути статистику: **кількість оренд**, **рейтинг орендаря** та **рейтинг власника**, поточний **статус верифікації** (не верифіковано / на розгляді / верифіковано / відхилено).

## 2.13. Відгуки та система довіри

Після завершення оренди обидві сторони можуть залишити **двосторонній відгук** — власник оцінює орендаря, а орендар власника. Відгуки впливають на **Trust Score** (рейтинг довіри), який відображається на сторінці речі та у профілі. Система довіри також включає **верифікацію особистості** та накопичувану **кількість успішних оренд**.

## 2.14. Адміністративна панель

Доступна лише користувачам із роллю **ADMIN** (пункт **Адмін** у навігації). Дозволяє:

- Переглядати та модерувати **оголошення** (схвалення речей);
- Керувати **користувачами** (зокрема обробляти запити на верифікацію, блокування);
- Переглядати **бронювання**;
- Бачити **статистику платформи** (кількість користувачів, активні оренди, популярні категорії).

# Додаток. Типові проблеми

| Симптом | Можлива причина та рішення |
|---|---|
| Фронтенд не отримує дані, помилки в консолі | Не запущено бекенд або БД. Перевірте `docker compose ps` і `./gradlew bootRun`. |
| Бекенд не стартує, помилка підключення до БД | Не піднято контейнер PostgreSQL або невірні `DB_*` у `backend/.env`. |
| Не працює вхід через Google | Не заповнені `OAUTH2_CLIENT_ID` / `OAUTH2_CLIENT_SECRET` у `.env`. |
| AI-помічник повертає помилку | Не заданий `OPENAI_API_KEY` у `.env`. |
| Порт `8080` або `5173` зайнятий | Зупиніть процес, що використовує порт, або змініть порт (`SERVER_PORT` для бекенду). |
